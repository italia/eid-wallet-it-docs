## Summary of Changes in ARF 2.7.3

### Structural Improvements:

The requirements have been reorganized into clear sections:
- **Section A**: Support for WUA Use Cases (WUA_01 - WUA_08)
- **Section B**: WUA in relation to Cryptographic Keys (WUA_09 - WUA_16)
- **Section C**: Requirements regarding privacy (WUA_17 - WUA_18)
- **Section D**: Miscellaneous requirements (WUA_19 - WUA_21)

### Clarifications to Empty Requirements:

Multiple placeholder requirements marked with "?" have been clarified as "Empty":
- **WUA_04**: ? → Empty
- **WUA_10**: ? → Empty (with note about move to Topic 18)
- **WUA_13**: ? → Empty
- **WUA_14**: ? → Empty (with note about move to Topic 18)
- **WUA_15**: ? → Empty
- **WUA_19**: ? → Empty
- **WUA_21**: ? → Empty

### Reference Standardization:

**WUA_20a**: Updated to reference Technical Specification 3 directly instead of vague "the technical specification mentioned in WUA_19", improving clarity and making the requirement self-contained.

### Key Notes:

1. **Document Status**: Technical Specification 3 is not yet finalized - discussions on WUA are ongoing
2. **Usage Restriction**: WUA will likely only be sent to PID/Attestation Providers during issuance, not to Relying Parties
3. **Topic Movements**: Some requirements (WUA_10, WUA_14) have been moved to Topic 18 on combined presentations

### Overall Assessment:

The WUA requirements have undergone organizational improvements and clarification of empty/placeholder requirements. The core functional requirements (WUA_01-WUA_09, WUA_11-WUA_12, WUA_16-WUA_18) remain substantively unchanged, maintaining stability in the essential WUA functionality while improving documentation clarity.
