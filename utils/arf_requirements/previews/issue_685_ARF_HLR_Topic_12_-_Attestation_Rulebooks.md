## Summary of Changes in ARF 2.7.3

### Key Modifications:

**ARB_05**: Updated OpenID4VP terminology - changed from "Input Descriptor" in "Presentation Request" to "Credential Query" in "presentation request" (also standardized capitalization).

**ARB_06b**: Added third option for claim names - "or is a Private Name specific to the attestation type" - providing more flexibility for attestation-specific claims.

**ARB_29**: Minor corrections:
- Fixed typo: "he" → "the"
- Changed "is as described in" to "follow the descriptions in" for better clarity

### New Requirements:

**ARB_30** [NEW]: Mandatory specification of selective disclosure for SD-JWT VC claims
- Schema Providers MUST specify for ALL claims whether they MUST, MAY, or MUST NOT be selectively disclosable
- Includes detailed notes on non-selectively disclosable claims, grouped disclosures, and differences from ISO/IEC 18013-5

**ARB_31** [NEW]: Recommendation for Type Metadata Documents
- Schema Providers SHOULD define Type Metadata Documents (per Chapter 6 of SD-JWT VC)
- Should include Claim Selective Disclosure Metadata (Section 9.3 of SD-JWT VC)
- Provides advantages for developers, providers, RPs, and Wallet Units

**ARB_32** [NEW]: Recommendation for JSON Schemas
- Schema Providers SHOULD define JSON Schemas (per Section 6.5 of SD-JWT VC)
- Should be included/referenced in Type Metadata Document
- References validation requirements in Section 6.5.2 of SD-JWT VC

### Overall Assessment:

The ARB requirements have been significantly enhanced with three new requirements (ARB_30-32) that provide detailed guidance for SD-JWT VC attestations, particularly around:
1. **Selective disclosure control** - Mandatory specification of which claims can/must/cannot be selectively disclosed
2. **Type Metadata Documents** - Structured metadata for better interoperability
3. **JSON Schemas** - Formal schema definitions for validation

These additions reflect the maturation of SD-JWT VC as a format and provide Schema Providers with clearer guidance on creating well-defined, interoperable attestation types. The modifications to existing requirements (ARB_05, ARB_06b, ARB_29) are minor refinements for accuracy and clarity.
